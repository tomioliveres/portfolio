Caso profesional

Sistema de Gestión Operativa y Económica de Obras

Desarrollé un sistema interno para el seguimiento operativo y económico de proyectos de movimiento de suelos.

La plataforma centraliza información clave de obra, productividad, avances, certificaciones y cobros pendientes, facilitando el control diario y la toma de decisiones. El objetivo fue reemplazar registros dispersos en planillas por una herramienta web más clara, ordenada y trazable.

El sistema incluye dashboard de indicadores, gestión de obras activas e históricas, pantallas de detalle económico-operativo, seguimiento de avances por ítems y una vista mobile orientada a la carga desde campo.

  • Presupuestos por ítems, separando contrato base y adicionales.
  • Métricas físico-financieras calculadas en backend.
  • Autenticación y permisos por rol para acciones sensibles.

Por motivos de confidencialidad, el proyecto se presenta como caso de estudio anonimizado. Las capturas contienen datos ficticios y no exponen información real ni código propietario.

  • Next.js
  • React
  • TypeScript
  • Prisma
  • PostgreSQL
  • Tailwind CSS
  • Recharts
  • Playwright

Sistema de Estacionamiento

Aplicación de consola en C++ para administrar la operación completa de un estacionamiento: ingresos, reservas, cobros, tarifas y reportes. Modela entidades de negocio y persistencia local mediante archivos binarios.

  • Separa módulos para clientes, vehículos, pagos, tarifas y reportes operativos.
  • Resuelve persistencia local sin base de datos mediante archivos binarios.

Catálogo de Productos

Aplicación desktop en C# para gestionar artículos, marcas y categorías con filtros, búsqueda avanzada e imágenes locales o por URL. Desarrollada con arquitectura por capas y persistencia en SQL Server.

  • Incluye ABM completa de artículos con soporte de marcas y categorías relacionadas.
  • Permite filtros y búsqueda avanzada para operar sobre catálogos más grandes.
  • Soporta imágenes remotas y locales integradas a una UI desktop en WinForms.
